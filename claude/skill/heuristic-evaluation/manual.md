# ヒューリスティック評価 → Figmaコメント投稿 マニュアル

Claudeでヒューリスティック評価を実施し、その結果をFigmaの該当アートボードにコメントとして投稿するための手順書です。

---

## 事前準備

### 1. スクリーンショットとアートボード名を揃える

スクリーンショットのファイル名と、Figmaのアートボード名を**完全に一致**させておきます。

```
screenshot/
├── 01_LINE公式アカウントプロフィール画面.png
├── 02_LINE公式アカウントチャットメニュー.png
└── ...

Figma（スクショページ内）
├── 01_LINE公式アカウントプロフィール画面
├── 02_LINE公式アカウントチャットメニュー
└── ...
```

**命名規則：**
- 先頭に2桁の連番（`01_`, `02_`...）
- スペースの代わりに `_`（アンダースコア）
- 日本語OK

---

### 2. Figma APIトークンを取得する

1. Figmaを開く
2. 左上のアイコン → **Settings** → **Security**
3. **Personal access tokens** → **Generate new token**
4. 以下のスコープにチェックを入れる

| スコープ | 権限 |
|---|---|
| File content | Read |
| Comments | Write |

5. 生成されたトークンをコピーして保存（画面を閉じると二度と見られません）

---

### 3. FigmaファイルキーをURLから確認する

FigmaのURLからファイルキーを取得します。

```
https://www.figma.com/design/ENJ3BdsBS1nO7BEgGI5EUO/...
                              ^^^^^^^^^^^^^^^^^^^^^^
                              これがファイルキー
```

## 実施手順

### Step 1：トークンをVS Codeターミナルに保存する

VS Codeのターミナル（ `` Ctrl + ` `` で開く）で実行します。

```bash
echo "取得したトークンをここに貼り付け" > /tmp/figma_token.txt
```

保存できたか確認：
```bash
cat /tmp/figma_token.txt | wc -c
# 0以外の数字が表示されればOK
```

> ⚠️ トークンはチャットに貼り付けないでください。ターミナルのみで扱います。

---

### Step 2：ローカルのスクリーンショットを確認してから評価を依頼する

**最初にローカルのスクリーンショットフォルダを確認する（必須）。**
Figma APIの画像だけに頼ると、取得漏れが発生するリスクがある。

```bash
ls screenshot/
```

ファイル一覧を確認した上で、以下のプロンプトをClaude Codeに入力します。

```
screenshot/フォルダにあるスクリーンショットを確認して、
評価軸H1〜H10に基づいてヒューリスティック評価を実施してください。
各画面ごとに以下の形式でまとめてください：

- 画面名
- 違反する評価軸（H番号）
- 問題の説明
- 深刻度（1〜4）
- 改善案
```

---

### Step 3：Figmaのアートボード一覧を取得する

以下をClaudeに依頼します。

```
FigmaファイルキーはENJ3BdsBS1nO7BEgGI5EUOです。
/tmp/figma_token.txt のトークンを使って、
アートボードの一覧とnode_idを取得してください。
```

**重要：`depth=5` を指定する**
`depth=3`（デフォルト）ではネストが深いフレームが取得できない。必ず `?depth=5` を使用すること。

```bash
curl -s "https://api.figma.com/v1/files/{key}/nodes?ids={page_id}&depth=5" \
  -H "X-Figma-Token: $(cat /tmp/figma_token.txt)"
```

Claudeが以下のようなリストを返します：

```
[31:2]  01_LINE公式_本人確認依頼メッセージ
[10:431] 02_Googleフォームログイン_メールアドレス入力_空
...
```

---

### Step 4：評価結果をFigmaにコメント投稿する

以下をClaudeに依頼します。

```
取得したnode_idと評価結果を照合して、
各アートボードにヒューリスティック評価のコメントを投稿してください。
```

**重要ルール：1コメント = 1問題**
複数の問題を1つのコメントにまとめないこと。問題の数だけコメントを投稿する。

> ⚠️ 同じアートボードに複数コメントを投稿すると、Figma上では全て同じ位置に重なって表示される。これはFigmaの仕様上避けられないため、Notionのチケット一覧から全問題を把握できるようにしておくことが重要。

コメントの形式：

```
問題:
[問題の説明]

解決策:
[改善案の説明]

--
ヒューリスティック評価
H[番号] [評価軸名]｜深刻度 X（意味）
```

> ⚠️ **Figmaコメントでは `*text*` は太文字にならず、そのまま `*` が表示される。**
> Figmaコメント APIはプレーンテキストのみのため、マークダウン記法は使用しない。
> `問題:` / `解決策:` はラベルとしてそのまま記載する。

深刻度の意味：

| 深刻度 | 意味 |
|---|---|
| 1 | 気になる程度 |
| 2 | 軽度の問題 |
| 3 | 重大な問題 |
| 4 | 致命的 |

**Figma API 技術仕様（コメント投稿時）：**
- `client_meta` には必ず `"node_offset": {"x": 0, "y": 0}` を含める
- レート制限（HTTP 429）が発生したら10秒待機後、リクエスト間に2秒の遅延を入れて再試行する

## 評価軸一覧（H1〜H13）

| ID | 評価軸 | 問いかけ |
|---|---|---|
| H1 | 今どこにいるかわかるか | 予約状況・支払い状況・認証の進捗が常に伝わるか |
| H2 | 言葉やアイコンが自然か | スポーツ・予約の文脈に合った表現が使われているか |
| H3 | 戻れる・やり直せるか | キャンセル・修正・前の画面に戻る手段があるか |
| H4 | 画面間でバラつきがないか | ボタンや表現のスタイルが全体で統一されているか |
| H5 | 間違えにくいか | 誤タップ・誤入力が起きにくい設計になっているか |
| H6 | 覚えなくていいか | 前の画面の内容を記憶しなくても進められるか |
| H7 | サクサク進めるか | 無駄なステップなく目的の操作にたどり着けるか |
| H8 | 画面がスッキリしているか | 余分な情報や要素が詰め込まれていないか |
| H9 | エラーから立ち直れるか | 何が問題で、どうすれば解決できるかが伝わるか |
| H10 | 迷ったとき助けてもらえるか | 必要な場所にガイドや説明が置かれているか |

### Step 5：NotionにチケットとしてFigmaコメントURLを記録する

Figmaコメント投稿後、各問題をNotionのチケットとして登録します。

**必須：FigmaコメントURLをNotionチケット内に含める**

FigmaコメントのURLは以下の形式で構成します：
```
https://www.figma.com/design/{file_key}?node-id={node-id-with-dashes}#comment-{comment_id}
```

※ `node-id` はFigmaのnode_id（`123:456` 形式）をダッシュ区切り（`123-456`）に変換する。

Notionチケットの内容テンプレート：

```markdown
## 基本情報
| 項目 | 内容 |
|------|------|
| 対象画面 | [画面名] |
| 評価軸 | H[X] [評価軸名] |
| 深刻度 | [X]（[意味]） |
| Figmaコメント | [コメントを見る](https://www.figma.com/design/...) |

## 問題
[問題の説明]

## 改善案
- [改善案1]
- [改善案2]
```

---

## よくあるエラー

| エラー | 原因 | 対処 |
|---|---|---|
| `Invalid token` | トークンが正しくコピーできていない | トークンを再生成して保存し直す |
| `Invalid scope` | トークンのスコープが足りない | File content: Read と Comments: Write を両方チェックして再生成 |
| コメントが見えない | 見ているFigmaのページが違う | 「スクショ」ページに移動して確認する |

---

## 注意事項

- `/tmp/figma_token.txt` はPCを再起動すると消えます。毎回保存が必要です
- トークンはチームメンバーと共有せず、各自で発行してください
- Figmaのアートボード名を変更した場合は、スクリーンショットのファイル名も合わせて変更してください
