# ヒューリスティック評価 → Figmaコメント投稿 スキル

## 評価軸一覧（H1〜H10）

| ID | 評価軸 | 問いかけ |
|---|---|---|
| H1 | 今どこにいるかわかるか | ユーザーの進捗が常に伝わるか |
| H2 | 言葉やアイコンが自然か | 行動や状態の文脈に合った表現が使われているか |
| H3 | 戻れる・やり直せるか | キャンセル・修正・前の画面に戻る手段があるか |
| H4 | 画面間でバラつきがないか | ボタンや表現のスタイルが全体で統一されているか |
| H5 | 間違えにくいか | 誤タップ・誤入力が起きにくい設計になっているか |
| H6 | 覚えなくていいか | 前の画面の内容を記憶しなくても進められるか |
| H7 | サクサク進めるか | 無駄なステップなく目的の操作にたどり着けるか |
| H8 | 画面がスッキリしているか | 余分な情報や要素が詰め込まれていないか |
| H9 | エラーから立ち直れるか | 何が問題で、どうすれば解決できるかが伝わるか |
| H10 | 迷ったとき助けてもらえるか | 必要な場所にガイドや説明が置かれているか |

## 深刻度スケール

| 深刻度 | 意味 | 対応優先度 |
|---|---|---|
| 0 | 問題なし | 対応不要 |
| 1 | 気になる程度・ほぼ影響なし | 低（余裕があれば対応） |
| 2 | 軽度の問題・一部ユーザーに影響 | 中（次回リリースで対応） |
| 3 | 重大な問題・多くのユーザーに影響 | 高（早急に対応） |
| 4 | 致命的・タスク完了を妨げる | 最高（即対応必須） |

## 事前準備

### 1. スクリーンショットとFigmaアートボード名を揃える

スクリーンショットのファイル名とFigmaのアートボード名を**完全に一致**させる。

```
screenshot/
├── 01_LINE公式アカウントプロフィール画面.png
├── 02_LINE公式アカウントチャットメニュー.png

Figma（スクショページ内）
├── 01_LINE公式アカウントプロフィール画面
├── 02_LINE公式アカウントチャットメニュー
```

命名規則：先頭2桁の連番（`01_`）、スペース不可（`_`使用）、日本語OK

### 2. Figma APIトークンの取得

1. Figma → Settings → Security → Personal access tokens → Generate new token
2. スコープ：`File content: Read` と `Comments: Write` の両方をチェック
3. 生成されたトークンをコピー（画面を閉じると二度と見られない）

### 3. FigmaファイルキーをURLから確認

```
https://www.figma.com/design/ENJ3BdsBS1nO7BEgGI5EUO/...
                              ^^^^^^^^^^^^^^^^^^^^^^
                              これがファイルキー
```

## 実施手順

### Step 1：トークンをターミナルに保存

```bash
echo "取得したトークンをここに貼り付け" > /tmp/figma_token.txt
cat /tmp/figma_token.txt | wc -c  # 0以外ならOK
```

> ⚠️ トークンはチャットに貼り付けない。ターミナルのみで扱う。

### Step 2：ローカルのスクリーンショットを確認してから評価を依頼

```bash
ls screenshot/
```

ファイル一覧確認後、以下のプロンプトをClaudeに入力：

```
screenshot/フォルダにあるスクリーンショットを確認して、
評価軸H1〜H10に基づいてヒューリスティック評価を実施してください。
各画面ごとに以下の形式でまとめてください：

- 画面名
- 違反する評価軸（H番号）
- 問題の説明
- 深刻度（1〜4）
- 改善案
```

### Step 3：Figmaのアートボード一覧を取得

```
FigmaファイルキーはXXXXXXXXXXXXです。
/tmp/figma_token.txt のトークンを使って、
アートボードの一覧とnode_idを取得してください。
```

**重要：`?depth=5` を指定する**（`depth=3`ではネストが深いフレームが取得できない）

```bash
curl -s "https://api.figma.com/v1/files/{key}/nodes?ids={page_id}&depth=5" \
  -H "X-Figma-Token: $(cat /tmp/figma_token.txt)"
```

### Step 4：評価結果をFigmaにコメント投稿

```
取得したnode_idと評価結果を照合して、
各アートボードにヒューリスティック評価のコメントを投稿してください。
```

**コメントフォーマット（1コメント = 1問題）：**

```
問題:
[問題の説明]

解決策:
[改善案の説明]

--
ヒューリスティック評価
H[番号] [評価軸名]｜深刻度 X（意味）
```

> ⚠️ FigmaコメントAPIはプレーンテキストのみ。`*text*` などのマークダウン記法は使用しない（`*` がそのまま表示される）。

**Figma API 技術仕様：**
- `client_meta` に必ず `"node_offset": {"x": 0, "y": 0}` を含める
- レート制限（HTTP 429）発生時：10秒待機後、リクエスト間に2秒の遅延を入れて再試行

### Step 5：NotionにチケットとしてFigmaコメントURLを記録

FigmaコメントURL形式：
```
https://www.figma.com/design/{file_key}?node-id={node-id-with-dashes}#comment-{comment_id}
```
※ `node-id` は `123:456` → `123-456`（コロンをダッシュに変換）

Notionチケットテンプレート：

```markdown
## 基本情報
| 項目 | 内容 |
|------|------|
| 対象画面 | [画面名] |
| 評価軸 | H[X] [評価軸名] |
| 深刻度 | [X]（[意味]） |
| Figmaコメント | [コメントを見る](https://www.figma.com/design/...) |

## 問題
[問題の説明]

## 改善案
- [改善案1]
- [改善案2]
```

## よくあるエラー

| エラー | 原因 | 対処 |
|---|---|---|
| `Invalid token` | トークンが正しくコピーできていない | 再生成して保存し直す |
| `Invalid scope` | スコープが足りない | File content: Read と Comments: Write を両方チェックして再生成 |
| コメントが見えない | 見ているFigmaのページが違う | 「スクショ」ページに移動して確認 |

## 注意事項

- `/tmp/figma_token.txt` はPC再起動で消える。毎回保存が必要
- トークンはチームメンバーと共有せず、各自で発行する
- Figmaのアートボード名を変更した場合、スクリーンショットのファイル名も合わせて変更する
