# SEOコラム記事 ライティングフレームワーク

## Description
各STEPで「何をどう考えるか」を定義する知識ベース。進行・確認・ファイル生成は `/seo-article` コマンドが管理する。

---

## Core Principles（基本原則）

- **アンサーファースト**: 結論を最初に示す。読者が「自分の悩みへの答えがここにある」と瞬時に判断できる構成を優先する
- **検索意図への忠実性**: タイトル〜本文〜CTAまで、検索意図から外れない一貫した設計をする
- **検索カニバリ防止**: 同サイト内に類似記事がある場合、意図が被っていないか確認する。意図が同じなら既存記事の拡充を検討する
- **独自性の確立**: 競合が提供していない視点・情報・深度を必ず1つ以上盛り込む

---

## STEP 1: キーワード理解（周辺KWの網羅）

検索KWを受け取ったら、まず以下を洗い出す。

1. **メインKW** の検索意図を分類する
   - Know型（情報収集）/ How to型（方法を知りたい）/ Buy型（購入・申込）/ Go型（特定サイトへ到達）
2. **関連KW・サジェスト・共起語** を展開し、ユーザーが同時に気になるであろうテーマを列挙する
3. **周辺KWから見出しMUST/WANTを仮設定** する（後のペルソナ設定・競合調査で精査）

> 目的：網羅性の確保。記事が「そのKWを検索した人のあらゆる疑問に答えられる」状態を目指す

---

## STEP 2: ペルソナ設定

以下の項目を埋め、1人の具体的な人物像を定義する。
複数パターンが想定される場合は2〜3パターン設定してよいが、**優先順位を必ずつける**（CVに近い・マジョリティなどの観点で）。

| 項目 | 内容 |
|------|------|
| 名前（仮名） | |
| 年齢 | |
| 性別 | |
| 業界と職種 | |
| 雇用形態・立場 | |
| 知識レベル | 初心者 / 中級者 / 上級者 |
| 検索のタイミング | ユーザージャーニーのどのフェーズか |
| その他（性格・転職回数など） | |

### インテンションの把握
> ペルソナが「知りたいこと」「感じているハードル・不安」は何か？
> 最重要な悩みを1文で言語化する。

### インサイトの把握
> なぜそれを知りたいのか？本当に解決したいことは何か？
> 表面的な検索意図の奥にある動機を言語化する。

## STEP 3: 競合調査（勝ち筋の特定）

| 順位 | | | |

### 競合の弱点分析（＝自社が勝てるポイント）

以下の観点で競合の弱点を特定し、差別化ポイントを導き出す。

| 観点 | 競合の現状の弱点 | 自社の勝ち筋 |
|------|-----------------|-------------|
| 具体性 / 実務再現度 | | |

> 目的：競合に勝つために、どこに独自性を発揮するかを明確にする

---

## STEP 4: 網羅性の確保

### 定量調査
- サジェスト・共起語・People Also Ask（ほかの人はこちらも質問）などを活用
- 「この記事に入れるべきテーマ」を漏れなく洗い出す

### 定性調査（必要に応じて）
- Q&Aサイト（Yahoo知恵袋・Reddit等）を読み込み、定量調査で拾えないニーズを補完
- インタビューが可能な場合は実施する

### 見出しテーマの決定

**見出しMUSTで扱うテーマ**（必ず見出しに入れるテーマ）
- → 検索意図・インテンション・定量調査から導出

**見出しWANTで扱うテーマ**（できれば見出しに入れたいテーマ）
- → インサイト・定性調査・独自性から導出

---

## STEP 5: メッセージ・一貫性の設計

### 記事のメインメッセージ（ゴール）
> 「〇〇というユーザーに〇〇というキッカケを提供する」の形式で記事を一言で表す
> ユーザーはこの記事を読むことで何が得られるのか？

### サービス・商品の紹介フロー
> どのサービス・会社を、どんな流れで紹介するか
> 例：〇〇軸の比較表を作り、上位3社（A社・B社・C社）の中から最適なものを選ばせる

### インサイトと強みの結節点
> ユーザーのインサイト × サービス・会社の強み の交点はどこか？
> どの強みをどう訴求して、ユーザーをどこへ誘導するか
> 例：リボ地獄に陥るのは高い手数料のせい × 任意整理ならその手数料がカットできる

---

## STEP 6: 見出し構成の設計

以下の順序で見出しを設計する。

1. **H1（タイトル）** ：KWを含め、ユーザーの検索意図に直接応える形で
2. **H2（大見出し）** ：MUSTテーマを軸に設計。論理的な流れで並べる
3. **H3（中見出し）** ：各H2を詳細化。ユーザーの「?」を「!」に変える単位

### アンサーファースト構成の原則
- 記事冒頭でメインKWに対する答えを先出しする（BLUF：Bottom Line Up Front）
- 結論 → 理由・根拠 → 詳細 → 応用の流れを意識する
- 読者が「自分の知りたい情報がある」と冒頭30秒で判断できるよう設計する

---

## STEP 7: 導入文（ファーストビュー）の設計

### 導入文の型

| # | 役割 | 要素 | 接続詞 | ポイント |
|---|------|------|--------|---------|
| 1 | 共感 | 問題提起 | ー | ペルソナが抱える悩み・不安をストレートに示す |
| 2 | 興味 | 解決策の提示（アンサーファースト） | じつは / そこで | 解決策を先に提示。読者に「答えがここにある」と感じさせる |
| 3 | 注意喚起 | インサイトの喚起 | ただし | インサイトを意識させ、続きを読みたくさせる |
| 4 | 期待 | 解決策が有効な理由 | なぜなら | 根拠・実績・エビデンスを示して信頼を獲得する |
| 5 | 目的意識 | ゴールの提示 | 〜なら〜を選びましょう | 読者が得られる結果をストレートに伝える |
| 6 | 把握 | コンテンツの要約 | ここでは | 記事全体の概要を示し、読み進める動機を作る |

> **導入文作成前に必ずインテンションとインサイトを設定すること**

## STEP 8: 品質チェック（6つの品質基準）

記事完成後、以下の基準で品質を評価する。

### 1. 網羅性
- ペルソナのニーズを定量・定性の両面で網羅できているか
- How to型 / Know型 / Buy型 / Go型に合った内容を盛り込めているか
- 網羅の幅・深さが適切か（広すぎず、浅すぎず）

### 2. 一貫性
- タイトル〜アイキャッチ〜導入文〜本文〜CTAのストーリーが一貫しているか
- 読者の感情曲線を意識し、「?」→「!」の対話が成立しているか
- 段落間のつながりが自然か

### 3. 助動性（背中押し）
- 読者の「?」を「!」に変えるだけでなく、次のアクションへの背中押しができているか
- CTAの位置・内容・数は適切か。広告感が強すぎず、弱すぎもしないか

### 4. 信頼性（専門性・権威性含む）
- データは適切に獲得・提示できているか（調査方法・サンプル数・グラフ選択など）
- 記事「ならでは」の専門性が実現できているか（他のKWに差し替えても成立しないか確認）
- 監修者・著者の権威性は適切に表現されているか

### 5. 可読性
- ペルソナの知識レベルに合った専門用語の難易度に設定できているか
- 文字量はペルソナの前のめり感に応じた適切な量か
- 図版・表・グラフで複雑な情報を視覚化できているか

### 6. 独自性
- 競合では得られない独自情報を盛り込めているか
- 独自性がインサイトへの訴求・背中押しとして機能しているか

### 7. 最新性
- 掲載情報は最新か（制度・キャンペーン・サービス内容など）

### 8. 時事性（記事による）
- 関連するトレンド・ニュースを適切なボリュームで盛り込んでいるか

---

## 検索カニバリ防止チェック

- 同サイト内に同一・類似の検索意図を持つ記事が存在しないか確認する
- 意図が被る記事がある場合は新規作成せず、既存記事の拡充を推奨する
- KWは異なっても意図が同じならカニバリになるため注意する
